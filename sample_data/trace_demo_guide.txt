File Formats Supported by Trace for Document Upload
Trace supports the following file formats for document upload: PDF, DOCX, TXT, MD, JSON, CSV, HTML, HTM. The maximum upload size per file is 20 megabytes. When you upload a document Trace parses the file content and splits it into chunks of up to 512 characters with a 64 character overlap between consecutive chunks. Each chunk is embedded using the Gemini embedding API and indexed in ChromaDB for fast retrieval. Only owners of a document can delete them and only uploaded documents are used as context sources for chat answers.

Confidence Score Calculation in Trace
The overall confidence score in Trace combines four weighted components. The first component is the relevance score which contributes 35 percent of the total weight. The second component is semantic similarity which contributes 25 percent of the total weight. The third component is source quality which contributes 20 percent of the total weight and measures both document diversity across multiple sources and score consistency across retrieved chunks. The fourth component is the support ratio which contributes 20 percent of the total weight and measures how many of the top five retrieved chunks have a score above 0.3 out of 5 total. Confidence labels are assigned based on the overall score: high confidence for scores at or above 0.75 and medium confidence for scores between 0.45 and 0.75 and low confidence for scores below 0.45.

Authentication and Security in Trace
Trace uses JSON Web Token JWT authentication with bcrypt password hashing for user security. Each user has a private isolated workspace where uploaded documents and conversation history are completely inaccessible to other users. Authentication tokens expire after 1440 minutes which is 24 hours. The system automatically redacts personally identifiable information PII from chat messages before they are stored. Rate limiting restricts API usage to 30 requests per minute per user account. The backend uses SQLite as the primary database with asynchronous database access via the aiosqlite library.

Hybrid Retrieval Architecture of Trace
Trace uses a hybrid retrieval system that combines dense vector search with sparse keyword matching for finding relevant document chunks. The dense retrieval component converts text chunks into high dimensional vector representations using Gemini embedding models and measures semantic similarity between the query and each chunk. The sparse retrieval component uses TF-IDF with unigram and bigram text features and English stop word removal and sublinear term frequency scaling for keyword based matching. The final combined score for each chunk is computed as 60 percent of the dense similarity score plus 40 percent of the sparse similarity score. A TF-IDF based reranker then reorders the top results using cosine similarity between the query and each chunk text.

LLM Providers Used by Trace
Trace supports two language model providers for generating chat answers. The Gemini provider from Google uses models like gemini-2.0-flash and requires a GEMINI_API_KEY environment variable to be configured. The Groq provider uses models like llama-3.3-70b-versatile and requires a GROQ_API_KEY environment variable to be configured. The LLM provider is selected by setting the LLM_PROVIDER environment variable to either gemini or groq. The LLM temperature is configured at 0.3 for consistent factual responses and the maximum output tokens is set to 1024 tokens per response. Content moderation is enabled by default.

API Endpoints in Trace
The Trace backend exposes REST API endpoints for several features. The auth endpoints handle user registration at POST api auth register and user login at POST api auth login. The document endpoints handle file upload at POST api documents upload and file deletion at DELETE api documents document id and file listing at GET api documents. The chat endpoints handle sending a message at POST api chat and streaming a message at POST api chat stream and conversation listing at GET api conversations and conversation deletion at DELETE api conversations conversation id. The feedback endpoint accepts ratings at POST api feedback. The health check is at GET api health.

Frontend Interface of Trace
The Trace frontend is built with React and Vite and uses a glassmorphism design style with frosted glass effects. The chat interface displays streaming responses character by character as they arrive. Citations appear as numbered links inline in the answer text and clicking a citation scrolls the source panel to show the exact document chunk that was used. A confidence gauge shows the overall confidence score and the label of high or medium or low. Source cards display the document name and chunk content and relevance score for each source. Users can rate individual responses with thumbs up or thumbs down to provide feedback. The knowledge panel lists all uploaded documents. Suggested questions are shown after each answer to help guide further exploration. Conversation history is saved and displayed in the sidebar for quick navigation.
